01

What is Logistic Regression?

You already know Linear Regression — it predicts a number (like house price, temperature). But what if you want to predict a category? Like "Will this student pass or fail?" or "Is this email spam or not?"

That's where Logistic Regression comes in. Despite the name, it's a classification algorithm — it predicts which class/category something belongs to.

🧠 Think of it this way

Imagine a doctor saying: "There's a 90% chance this tumour is malignant." They're not predicting a number — they're predicting a probability of belonging to a class. That's exactly what Logistic Regression does.

Linear Regression

Predicts a continuous number.
e.g. "House price = ₹45 lakh"

Logistic Regression

Predicts a class/category.
e.g. "Pass (1) or Fail (0)"

02

The Sigmoid Function — The Heart of It

The problem with using Linear Regression for classification is that it can give you values like -5 or 150 — which don't make sense as probabilities.

We need output between 0 and 1. So we pass the linear output through a Sigmoid function, which squishes any value into the (0, 1) range — giving us a probability!

g(z) → 1  as z → +∞
g(z) → 0  as z → −∞
g(z) = 0.5 at z = 0
S-shaped curve → probabilities
g(z) = 1 / (1 + e−z)
where z = θ₀ + θ₁x₁  (your linear prediction)
💡 So the full hypothesis is:   hθ(x) = 1 / (1 + e−θᵀx)
This gives the probability that the input belongs to class 1. If it's > 0.5, predict class 1. If less, predict class 0.
03

Cost Function — Log Loss

In Linear Regression, we used MSE (Mean Squared Error) as the cost function. But if we use MSE with the sigmoid function, we get a bumpy, non-convex curve with many local minima — gradient descent gets stuck!

So instead, we use a smarter cost function called Log Loss (Binary Cross Entropy).

🧠 Why Log Loss?

Log loss heavily punishes confident wrong predictions. If the real answer is 0 but you predicted 0.9 (very confident and wrong), you get a huge penalty. But if you predicted 0.1 (slightly off), the penalty is small. This forces the model to be careful!

y = 0, ŷ = 0.9
😱 HUGE
Very wrong → very penalized
y = 0, ŷ = 0.1
✅ Small
Almost right → tiny penalty
J(θ) = −(1/m) Σ [ y·log(ŷ) + (1−y)·log(1−ŷ) ]
This gives a convex curve → guaranteed global minimum ✓
04

Training — Gradient Descent

Just like Linear Regression, we minimize the cost function using Gradient Descent. The math looks the same — only the hypothesis function changed (sigmoid instead of linear).

1
Initialize Parameters Start with θ₀ = 0, θ₁ = 0 (random or zeros)
2
Make Predictions Run all data through hθ(x) = sigmoid(θᵀx) to get probabilities
3
Compute Gradients ∂J/∂θ₁ = (1/m) · (ŷ − y) · x   |   ∂J/∂θ₀ = (1/m) · (ŷ − y)
4
Update Parameters θ = θ − α · gradient  (α = learning rate)
5
Repeat Until Convergence Keep going until the cost stops decreasing significantly
🎯 Key insight: The gradient formula  (ŷ − y)  looks identical to Linear Regression! The difference is only inside the prediction step where sigmoid is applied.

3 Types of Logistic Regression

🔵 Binomial

2 output classes only

Yes/No, Pass/Fail, Spam/Not spam
🟢 Multinomial

3+ classes, no order between them

Cat / Dog / Rabbit
🟡 Ordinal

3+ classes with a natural ranking

Low / Medium / High
06

Assumptions (When to Use It)

🔗
Independent Observations
Each data point should be unrelated to others. No duplicates, no correlated rows.
🎯
Binary Output (for basic Logistic)
Output must be 0 or 1. For 3+ classes, use Softmax instead.
📈
Linear Relationship with Log-Odds
Features should have a linear effect on the log-odds (log(p/1-p)). Non-linear? Consider feature engineering.
🚫
No Extreme Outliers
Outliers can heavily distort the coefficients and mess up the decision boundary.
📊
Large Enough Dataset
Small datasets make the model unstable. More data = more reliable results.
